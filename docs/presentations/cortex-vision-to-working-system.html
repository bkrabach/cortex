The Breakthrough

A Happy Accident

💡

During testing, an unrelated notification arrived 4 minutes before a meeting. The system dismissed it appropriately, but then - used the opportunity to remind me about the upcoming meeting.

This serendipitous behavior inspired the always-on scheduler design. What if Cortex didn't just react - but proactively surveyed state and took action?

Today

Phase 1 Complete

Triage System

Non-critical notifications no longer vanish. They accumulate for review at a good time.

User Experience

The Triage Triple

Every queued item gets one of three responses:

Deal with
Ignore
Already handled
Deal with
Opens pre-drafted responses and suggested actions. System learns this was important.
Ignore
Dismisses the item. System learns to suppress similar notifications in the future.
Already handled
Marks complete without affecting learning. You dealt with it outside Cortex.

Optional feedback on any action helps steer future decisions

Cortex Core

The Always-On Scheduler

CortexScheduler wakes up periodically, surveys state, and takes action.

15m

Default Wake-up

Check pending triage, upcoming calendar, domain expert state

1m

After Action

If something happened, wake sooner to check for follow-up

1h

When Idle

Nothing happening? Lengthen interval to save resources
